Модуль расстановки отвечает за присвоение квартире Х и выбранным уровням треков (`internal/configs/tracks.json` см. Управление треками) набора устройств с конкретными координатами и фильтрами. TODO: добавить скриншот плана пустой квартиры с подписью и с расставленными устройствами.

## Основные компоненты и их задачи
- `cmd/layout/main.go` - инициализация всех модулей и запуск программы
- `internal/events/engine/engine.go` - движок расстановки всех устройств в квартире
- `rules/storage/storage.go` - хранилище правил расстановки устройств

## Работа правил расстановки устройств

### Интерфейс Rule

Каждое устройство реализует интерфейс `Rule`, основным методом которого является `Apply`:
```go
type Rule interface {
	Type() string

	Apply(zonedAP *apartment.ZonedApartment, levelNum string, deviceRooms []string, maxCount int, layout *apartment.Layout) error
}
```

### Алгоритм работы

1. Для каждого нужного устройства вызывается свой метод `Apply`
2. Внутри анализируется планировка квартиры и определяется оптимальное место для устройства.
3. Результат сохраняется в `layout` в виде структуры `Placement`:
```go
type Placement struct {
	Device   *Device              `json:"device"`
	Position *point.Point         `json:"position"`
	Filters  filters.DeviceFilter `json:"filters,omitempty"`
}
```

Таким образом, сервис для каждой квартиры определяет (1) какие устройства ставить (включая тип этого устройства и возможные обязательные характеристики) и (2) его координаты.

**Примечание**: Поле `Filters` опционально, так как не для всех устройств требуются фильтры

## API

Сервис запускается на порту `8080` и предоставляет два HTTP эндпоинта:

- `/apartment` - загрузка планировки квартиры
- `/layout` - расстановка устройств и последующее отправление результата

## Управление треками

### Файл конфигурации
Путь: `internal/configs/tracks.json`

### Архитектура треков

Каждый трек представляет собой набор уровней, где каждый из них содержит:
- Базовую информацию (название, описание трека)
- Список устройств
- Дефолтное распределение устройств по комнатам
- Лимиты на количество устройств
- Фильтры устройств

### Добавление трека

Для добавления трека необходимо в файле `internal/configs/tracks.json` добавить новый ключ в объект `tracks`.

**Пример структуры трека:**

```json
{
  "tracks": {
    "security": {...},
    "new_track": {
      "name": "Название трека",
      "levels": {
        "1": {
          "devices": ["Типы устройств"],
          "device_rooms": {...},
          "max_device_counts": {...},
          "device_filters": {...}
        },
        "2": {...},
        ...
      }
    }
  }
}
```

**Примечание**: каждый последующий уровень дополняет предыдущий, то есть устройства наследуются при переходе на следующий уровень, однако остальные поля (`max_device_counts`, `device_filters`) могут меняться.

## Работа правил расстановки устройств
Для расстановки каждого устройства типа Х определяется интерфейс `Rule`, главным методом которого является `Apply`
```go
Apply(apartment *entities.Apartment) map[string]map[string]*entities.Placement
```
который по плану квартиры выдает зависимость `roomID -> deviceType -> devicePlacement` (по идентификатору комнаты и типу устройства получаем его расстановку в этой комнате).

`Placement` содержит основную информацию для расстановки выбранного типа устройства (фильтр и координату).

Каждое устройство будет реализовывать интерфейс `Rule`, что позволит расставить их во всей квартире (в нужных местах, в нужном количестве) с помощью своего метода `Apply`.

Таким образом, сервис для каждой квартиры опряделяет, (1) какие устройства ставить(вклюяая тип этого устройства и возможные обязательные характеристики) и (2) его координаты. 

## Добавление/изменение трека
Треки описываются в `internal/configs/tracks.json`. Для добавления нового трека требуется описать его структуру, ориентируясь на уже существующие треки (например, `security`), набор полей должен совпадать. Также при описании стоит учитывать, что каждый последующий уровень **дополняет** предыдущий.

Чтобы изменить поле трека, необходимо просто модифицировать его значение.
