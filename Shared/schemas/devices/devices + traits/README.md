Привет! Это описание подхода, который мы придумали для работы с JSON-схемами умных устройств (для умного дома).
Вместо того чтобы хранить кучу почти одинаковых файлов, мы решили сделать конструктор: определяем общие блоки (traits), а потом собираем из них финальные схемы для каждого типа устройств с помощью небольшого скрипта.

Зачем мы так сделали

Чтобы не повторять одни и те же поля в каждом файле — принцип DRY.

Чтобы было легко добавлять новые устройства: просто комбинируем существующие блоки.

Чтобы микросервисы могли брать только нужные им данные. Например, сервису для расчёта энергопотребления достаточно блока battery_operated.

Если общее поле меняется, правим его в одном месте, и у всех устройств оно обновляется автоматически.

Что лежит в репозитории

traits.json — библиотека компонентов. Здесь собраны все возможные блоки:

    -base — базовые поля, которые есть у любого устройства (тип, название, цена, экосистема, протоколы, артикул Яндекса).

    -climate — для климатической техники (температурный диапазон, режимы, скорость вентилятора и т.д.).

    -security — для охранных устройств (тип сигнализации, громкость, тип безопасности).

    -motorized — для устройств с мотором (шторы, жалюзи, ворота): тип управления, источник питания, материал.

    -lighting — для световых устройств (мощность, напряжение, срок службы, цветовые характеристики).

    -sensor — для датчиков (точность измерения).

    -battery_operated — для устройств на батарейках (уровень заряда, резервное питание, источник питания).

    -network — для сетевого оборудования (поддерживаемые протоколы, количество устройств, порты и т.п.).

devices.json — здесь описываются конкретные типы устройств. Для каждого указано:

    -Какие блоки (traits) в него входят.

    -Если есть уникальные поля, они добавляются в секцию extra.
