# Модуль расстановки устройств

Модуль расстановки отвечает за присвоение квартире Х и выбранным уровням треков (`internal/configs/tracks.json` см. Управление треками) набора устройств с конкретными координатами и фильтрами. TODO: добавить скриншот плана пустой квартиры с подписью и с расставленными устройствами.

## Основные компоненты и их задачи
`cmd/layout/main.go` - инициализация всех модулей и запуск программы

`internal/events/engine/engine.go` - движок расстановки всех устройств в квартире

`rules/storage/storage.go` - хранилище правил расстановки устройств

## Работа правил расстановки устройств

Для расстановки каждого устройства типа Х определяется интерфейс `Rule`, главным методом которого является `Apply`
```go
Apply(apartment *entities.Apartment) map[string]map[string]*entities.Placement
```
который по плану квартиры выдает зависимость `roomID -> deviceType -> devicePlacement` (по идентификатору комнаты и типу устройства получаем его расстановку в этой комнате).

`Placement` содержит основную информацию для расстановки выбранного типа устройства (фильтр и координату).

Каждое устройство будет реализовывать интерфейс `Rule`, что позволит расставить их во всей квартире (в нужных местах, в нужном количестве) с помощью своего метода `Apply`.

Таким образом, сервис для каждой квартиры опряделяет, (1) какие устройства ставить(включая тип этого устройства и возможные обязательные характеристики) и (2) его координаты. 

## Управление треками

### Файл конфигурации
Путь: `internal/configs/tracks.json`

### Архитектура треков

Каждый трек представляет собой набор уровней, где каждый из них содержит:
- Базовую информацию (название, описание трека)
- Список устройств
- Дефолтное распределение устройств по комнатам
- Лимиты на количество устройств
- Фильтры устройств

### Добавление трека

Для добавления трека необходимо в файле `internal/configs/tracks.json` добавить новый ключ в объект `tracks`.

**Пример структуры трека:**

```json
{
  "tracks": {
    "security": {...},
    "new_track": {
      "name": "Название трека",
      "levels": {
        "1": {
          "devices": ["Типы устройств"],
          "device_rooms": {...},
          "max_device_counts": {...},
          "device_filters": {...}
        },
        "2": {...},
        ...
      }
    }
  }
}
```

**Важно**: каждый последующий уровень дополняет предыдущий, то есть устройства наследуются при переходе на следующий уровень, однако остальные поля (`max_device_counts`, `device_filters`) могут меняться.

## Запуск тестов
Для запуска тестов нужно перейти из корневой директории в директорию layout и запустить тесты:
```bash
cd services/layout
go test -v ./tests/...
```

## Запуск программы
Для запуска программы необходимо также находиться в директории layout и запустить `main.go`:
```bash
go run cmd/layout/main.go
```
