# Инструкция к ядру эвалюатора (QualityEvaluator)

Класс `QualityEvaluator` — это математическое ядро модели оценки качества IoT-устройств. Код написан по принципу Data-Driven: он не содержит хардкода конкретных типов устройств, а динамически парсит правила нормализации из централизованной таксономии проекта (общего `device_types.json`).

## 1. Как работает логика (Основные методы)

При инициализации класс принимает общую схему таксономии и веса `weights=(w1, w2, w3)` (по умолчанию 30% / 40% / 30% для репутации, характеристик и протоколов).

* **`normalize(value, min_val, max_val, inverse)`**: Приводит физическую величину к шкале [0.0 .. 1.0]. Поддерживает инверсию (`inverse=True`) для характеристик, где минимальное значение является лучшим (например, уровень шума).
* **`eval_reputation(rating, count)`**: Реализует математическое сглаживание через Байесовское среднее. Эффективно подавляет накрученные рейтинги у товаров с малым количеством отзывов.
* **`eval_protocol(protocols)`**: Оценивает транспортный уровень на основе сетевой топологии (Mesh = 1.0, Star = 0.7). Выдает синергетические бонусы устройствам с мультипротокольной поддержкой.
* **`eval_specs(trait_keys, specs)`**: Динамический парсер характеристик. Читает веса и границы из центральной таксономии. Автоматически агрегирует оценки для устройств "2-в-1" (если передан массив `trait_keys`).

## 2. Формат входных данных

Метод `evaluate_device()` ожидает на вход словарь, сгенерированный парсером маркетплейсов в соответствии с глобальной схемой. 
**Обратите внимание:** Вместо единичного типа используется массив `eval_traits` для поддержки гибридных устройств.

```json
{
  "name": "Яндекс Станция Миди (Колонка + Хаб)",
  "price": 14990,
  "eval_traits": ["eval_media", "eval_infra"],
  "protocol": ["wifi", "zigbee", "matter"],
  "reviews": {
    "rating": 4.9,
    "count": 1250
  },
  "specs": {
    "max_child_devices": 128
  }
}
