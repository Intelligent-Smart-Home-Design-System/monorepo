# quality-calculator

Это сервис, который вычисляет качество для каждого устройства в золотом слое (таблица devices).
В devices есть nullable-поле quality - его будет заполнять сервис на основе характеристик/рейтинга/цены/отзывов...
В пайплайне построения каталога запускается после сервиса построения каталога.

## Usage

```bash
quality-calculator run
```

## Логирование

Сервис использует `structlog` для структурированных JSON-логов. Формат и уровень логирования настраиваются через переменные окружения:

| Переменная | Описание | Значение по умолчанию |
|------------|----------|-----------------------|
| `LOG_FORMAT` | Формат вывода логов: `json` или `console` | `json` |
| `LOG_LEVEL` | Уровень логирования: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL` | `INFO` |

Пример запуска с настройками по умолчанию (JSON, INFO):

```bash
quality-calculator run
```

Для разработки с читаемым выводом в консоль:

```bash
LOG_FORMAT=console LOG_LEVEL=DEBUG quality-calculator run
```

В JSON-режиме каждый лог — это JSON-строка с полями `event`, `level`, `timestamp`, `logger` и дополнительными контекстными полями. Это позволяет парсить логи в системах агрегации (ELK, Loki, Cloud Logging).
