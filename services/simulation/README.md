# Сервис симуляции умного дома

Сервис для дискретно-событийной симуляции систем умного дома. Позволяет моделировать взаимодействия между людьми, устройствами, датчиками и сценариями автоматизации.

## Основные возможности

- **Дискретно-событийное моделирование** с использованием библиотеки `simgo`
- **Физическое моделирование**: геометрия комнат, движение людей с учётом препятствий
- **Разнообразные устройства**: освещение, климат, безопасность, датчики, мультимедиа и др.
- **Датчики с радиусом действия**: камеры, датчики движения с зоной в виде радиуса
- **Система сценариев**: граф зависимостей между сущностями
- **WebSocket**: управление симуляциями в реальном времени
- **Проверка корректности**: детектирование циклических зависимостей, валидация данных

## Основные компоненты

### 1. Engine (Движок симуляции)

Центральный компонент, управляющий всеми аспектами симуляции. Использует библиотеку [simgo](https://github.com/fschuetz04/simgo) для дискретно-событийного моделирования.

**Ключевые методы:**
- `NewSimEngine` — создание нового движка
- `InitEntities и InitProcesses` — инициализация и запуск процессов сущностей
- `Step` — выполнение одного шага симуляции с входящими событиями

**Граф зависимостей:**
- Сущности могут триггерить другие сущности
- Циклические зависимости запрещены и проверяются при инициализации

### 2. Simulations
Компонент для связи частей сервиса и управления движками клиентов.

**Ключевые методы:**
- `Start` — инициализация симуляции
- `Tick` — выполнение шага симуляции конкретного движка
- `Stop` — остановка симуляции и освобождение ресурсов


### 3. Люди

Представляют людей в квартире с реалистичной физикой движения.

**Особенности:**
- люди движутся в многоугольных комнатах
- определение находится ли человек в комнате
- детектирование столкновений со стенами
- люди могут проходить через двери между комнатами
- люди триггерят датчики (движения, присутствия) и могут взаимодействовать с устройствами

### 4. Устройства

#### Базовое устройство
Реализует базовое поведение устройств, используется как встраивание при реализации новых устройств.

```go
type BaseDevice[T any] struct {
   enginePort engine.EnginePort
   inStore    simgo.Store[T]
   
   handler func(T) T
   
   ID        string   `json:"id"`
   Delay     float64  `json:"delay"`
   Receivers []string `json:"receivers"`
}
```

#### Информация об устройствах
- simulation/configs/entities.json - список устройств с их коротким текстовым описанием
- simulation/configs/dependencies.json - описание потенциальных зависимостей между устройствами

**Датчики с радиусом действия:**
- Камеры и датчики движения могут обнаруживать людей в радиусе
- Вычисляется расстояние от датчика до человека
- При вхождении/выходе человека из радиуса триггерятся устройства

### 5. Поле (Field)

Представляет геометрию помещений в квартире/доме.

- определение находится ли точка внутри многоугольника
- проверка пересечения радиуса датчика с человеком

### 6. Converter

Преобразует входящие DTO.

**Основные функции:**
- конвертация DeviceDTO в объекты устройств
- построение графа зависимостей
- преобразование информации о плане

### 7. WebSocket Manager
Управляет клиентами симуляции и реализует WebSocket upgrade при первоначальном соединении.

### 8. WebSocket Client

Обработчик WebSocket подключений для управления симуляциями.

**Функции:**
- Приём команд
- Отправка состояния симуляции в реальном времени
- Обработка ошибок

## Правила и логика работы

### 1. Жизненный цикл симуляции

```
┌─────────────────────────────────────────────────┐
│ 1. WebSocket сообщение: simulation:start        │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│ 2. Engine.Start()                               │
│    - Инициализация сущностей                    │
│    - Построение графа зависимостей              │
│    - Проверка циклических зависимостей          │
│    - Запуск процессов                           │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│ 3. Симуляция активна, ожидает events:tick       │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│ 4. Engine.Tick(inputs)                          │
│    - Обработка входящих событий (если есть)     │
│    - Распространение по графу зависимостей      │
│    - Выполнение шага симуляции (simgo)          │
│    - Отправка исходящих событий клиенту         │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│ 5. Повторить шаг 4 или получить stop            │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│ 6. stop                                         │
│    - Остановка симуляции                        │
│    - Освобождение ресурсов                      │
└─────────────────────────────────────────────────┘
```

### 2. Обработка событий

1. **Получение события** через WebSocket или от сущностей
2. **Распространение**:
   - Событие поступает на сущность по EntityID
   - Движок обрабатывает событие
   - Все зависимые сущности получают уведомление
3. **Выполнение**:
   - Триггеренные сущности выполняют свои действия
   - Возможны цепочки событий через граф зависимостей
4. **Обработанные события** отправляются клиенту через WebSocket


### 3. Движение людей и физика
**Определение находится ли точка внутри многоугольника**
- Используется для проверки в какой комнате находится человек

**Столкновения:**
- Детектирование столкновения с стенами
- Остановка у стены при попытке выхода за границы комнаты
- Проход через двери между комнатами

**Датчики с радиусом:**
- Вычисляется расстояние от датчика до человека
- Проверяется пересечение окружности датчика с позицией человека, учитывая стены
- События триггерятся при входе/выходе из зоны

### 5. Процессы сущностей

Каждая сущность может иметь асинхронный процесс.

Процессы выполняются параллельно и координируются `simgo` для дискретно-событийного моделирования.

### 6. Валидация данных

**При инициализации проверяется:**
- Все люди находятся внутри валидных комнат
- Все устройства имеют корректный ID
- Граф зависимостей не содержит циклов

## Запуск

```bash
# Клонирование репозитория
git clone <repo-url>
cd services/simulation

# Установка зависимостей
go mod download

# Сборка
go build -o bin/simulation cmd/simulation/main.go

# Запуск
./bin/simulation
```

Сервис запустится на порту `:8080`.

## Разработка

### Добавление нового типа устройства

1. Создать новую структуру в `internal/entities/devices/`
2. Реализовать интерфейс `Entity` (и `EntityWithProcess` если нужен процесс)
3. Добавить конвертацию в `converter.go`
4. Обновить тесты в `end_to_end_test.go`
